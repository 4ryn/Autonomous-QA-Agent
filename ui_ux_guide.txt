E-SHOP CHECKOUT UI/UX GUIDE
===========================

FORM VALIDATION RULES
---------------------

Customer Name Field:
- Field ID: customer-name
- Field Name: name
- Type: Text input
- Required: Yes
- Validation: Cannot be empty
- Error Message ID: name-error
- Error Text: "Name is required"
- Validation Trigger: On form submission

Customer Email Field:
- Field ID: customer-email
- Field Name: email
- Type: Email input
- Required: Yes
- Validation: Must match email format (contains @ and domain)
- Regex Pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/
- Error Message ID: email-error
- Error Text: "Valid email is required"
- Validation Trigger: On form submission

Shipping Address Field:
- Field ID: customer-address
- Field Name: address
- Type: Text input
- Required: Yes
- Validation: Cannot be empty
- Error Message ID: address-error
- Error Text: "Address is required"
- Validation Trigger: On form submission

Discount Code Field:
- Field ID: discount-code
- Field Name: discount-code
- Type: Text input
- Required: No
- Validation: Must be "SAVE15" (case-insensitive, converted to uppercase)
- Error Message ID: discount-error
- Error Scenarios:
  1. Empty field: "Please enter a discount code"
  2. Invalid code: "Invalid discount code"
- Success Behavior: Display alert "Discount code applied successfully!"

INTERACTION BEHAVIORS
---------------------

Cart Quantity Controls:
- Plus Button: Increases quantity by 1, updates prices immediately
- Minus Button: Decreases quantity by 1, updates prices immediately
- Remove Button: Removes item completely, updates prices immediately
- Quantity Display ID Pattern: quantity-{productId}
- All price updates happen automatically without page reload

Discount Code Application:
- Apply Button Click: Validates code and updates prices
- Success: Shows alert, applies 15% discount, updates all prices
- Failure: Shows error message below input field
- Error messages use inline display (not alerts)

Shipping Option Selection:
- Change Event: Updates shipping cost and total immediately
- Default Selected: Standard Shipping (Free)
- Radio buttons use name attribute for grouping

Payment Method Selection:
- Change Event: Selects payment method
- Default Selected: Credit Card
- Radio buttons use name attribute for grouping

Form Submission (Pay Now):
- Validates all required fields
- Shows inline error messages for invalid fields
- Shows alert for empty cart
- Shows alert for invalid form
- On success: Hides all sections, shows success message

ERROR MESSAGE DISPLAY
--------------------

Error Message Styling:
- Class: error-message
- Color: #f44336 (red)
- Font Size: 12px
- Display: Initially hidden (display: none)
- Shown: When validation fails (display: block)

Error Display Locations:
- Below each form input field
- Below discount code input field
- Alert dialogs for cart/form validation

USER FEEDBACK MESSAGES
----------------------

Alert Messages:
1. "Discount code applied successfully!" - When valid discount code applied
2. "Your cart is empty!" - When trying to checkout with empty cart
3. "Please fill in all required fields correctly." - When form validation fails

Success Message:
- Text: "Order placed successfully! Thank you for your purchase."
- Background: #4CAF50 (green)
- Display: Initially hidden, shown after successful payment
- Behavior: Hides all other sections when displayed

JAVASCRIPT FUNCTIONALITY
------------------------

Cart Management:
- Global cart object stores: {productId: {name, price, quantity}}
- addToCart(id, name, price): Adds or updates cart item
- removeFromCart(id): Removes item from cart
- updateQuantity(id, change): Modifies item quantity
- updateCart(): Refreshes cart display
- updatePrices(): Recalculates all prices

Discount Management:
- Global discountRate variable (0 to 1)
- applyDiscount(): Validates and applies discount code
- Updates all price displays immediately

Form Validation:
- validateForm(): Checks all required fields
- Returns boolean: true if valid, false if invalid
- Shows/hides error messages based on validation

Order Processing:
- processPayment(): Main checkout function
- Validates cart, form, then completes order
- Shows success message and hides all sections
